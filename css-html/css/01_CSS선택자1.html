CSS(Cascading Style Sheets)

            마크업 언어(html) 가 화면에 실제로 표시되는 방법을 기술하는 언어이다.
            (모양, 색상, 크기, 배경 이미지, 움직임 등등...)

            하나의 태그에 여러 css 코드를 겹쳐서 원하는 스타일을 만들어 낸다. 

            W3C에서 CSS를 표준 스타일로 지정해 두었다. 

            style 태그는 head 내부에 작성해도 되고,
                         body 내부에 작성해도 된다. 
                         심지어 head 와 body 사이에 존재하지 않아도 된다. 
                         하지만, head 내부에 작성해 주는 것이 개발자 간의 관례이며,
                         이는 추후 수정같은 추가 작업을 진행할 때 용이하도록 한 것이다.

                         단, id="변수명" 형식으로 표기할 경우에는 style 태그 내부에 #변수명{ } 의 형식으로

                    class 와 id 의 차이점
                              style과 관련된 코드를 작성할 때는, class 변수명 형식을 사용하는 것이 좋다. 
                         javascript와 관련된 코드를 작성할 때는, id 변수명 형식을 사용하는 것이 좋다. 
                         
                         class 는 여러 태그에 같은 스타일을 적용할 때 유용하다. 
                            id 보다 우선순위가 낮다. 
                            
                            id 는 특정한 기능을 가진 태그에 특정한 스타일을 유일하게 적용할 때 유용하다. 
                            특별하게 스타일을 가져야 하는 경우에 사용하기 때문에 우선순위가 높다. 

                        가능한 class 는 style 과 CSS 에 사용하고,
                        id 는 기능과 javascript 에 사용해 주는 것이 좋다.

                    2-4. 태그에 작성된 특정 속성을 사용하여 선택하는 선택자
                         input[name] : input 태그 내부에 존재하는 모든 name 속성 값에 공통된 디자인을 설정하고 싶을 때 사용한다. 
                         이 형식으로 작성할 경우, 맨 마지막에 작성한 디자인이 적용된다. 

                         만약, 같은 속성(지금은 name) 에서 name="변수명" 만 특별한 디자인을 갖게 하고 싶다면,
                         input[name="변수명"]{ } 형식으로 작성해 준 다음, 디자인을 적용하면 된다. 
                         
                         
                         

                3. .css 형식 파일을 사용하여, .css 파일 안에 css 코드 작성
                    head 태그 내부에 style 태그 대신 link 태그 사용
                    link 태그는 외부에 있는 css 파일과 연결하는 태그이다. 
                    (css 파일 이외에도 여러가지 형식의 파일을 여러개 연결 가능하다.)
                             rel(relationship) : 연결하고자 하는 파일이 어떤 형식인지 확인
                         href(hyper reference) : 참조하고자 하는 파일의 위치가 어디인지 확인(주소, 경로 등)

                    html 안에서 코드가 너무 길어지지 않게 하기 위해 사용한다. 
                    중복되는 코드를 하나의 파일 안에 작성해 두고, 여러 파일에서 사용할 수 있게 해준다.
